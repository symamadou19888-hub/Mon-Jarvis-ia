# Architecture des agents de Jarvis

## Objectif

Ce document définit l'organisation des agents qui composent Jarvis.

Chaque agent possède une spécialité précise et travaille en collaboration avec les autres agents sous la supervision du Directeur IA.

---

# Structure générale

Jarvis fonctionne comme une équipe d'agents spécialisés.

Le Directeur IA coordonne les missions et dirige les agents selon les besoins du projet.

---

# Directeur IA

Fichier :
00_directeur_ia.md

Rôle :
- Diriger l'ensemble du système.
- Comprendre les objectifs.
- Organiser les missions.
- Choisir les agents nécessaires.
- Vérifier la cohérence des résultats.

---

# Agent Développeur

Rôle :
- Concevoir les solutions techniques.
- Créer et améliorer le code.
- Étudier les architectures possibles.

---

# Agent Designer

Rôle :
- Concevoir les interfaces.
- Améliorer l'expérience utilisateur.
- Créer une identité visuelle.

---

# Agent Recherche

Rôle :
- Chercher des informations.
- Comparer les technologies.
- Analyser les tendances et les solutions existantes.

---

# Agent Business

Rôle :
- Étudier les opportunités.
- Réfléchir à la monétisation.
- Transformer les idées en produits utiles.

---

# Méthode de collaboration

1. L'utilisateur donne un objectif.
2. Le Directeur IA analyse la demande.
3. Les agents spécialisés travaillent sur leur partie.
4. Les résultats sont regroupés pour créer une solution complète.

---

# Vision

Créer une équipe d'agents IA capables de collaborer pour transformer des idées en projets réels.
