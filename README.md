# Memory - Système de mémoire de Jarvis

## Objectif

Le dossier `memory` contient toutes les informations que Jarvis doit conserver au fil du temps afin d'améliorer son efficacité, de suivre les projets et d'éviter de perdre des informations importantes.

La mémoire est séparée du cerveau (`brain`).

- Le cerveau contient les règles, les objectifs et les connaissances permanentes.
- La mémoire contient les informations qui évoluent avec le temps.

---

## Structure

### conversations/

Conserve les échanges importants avec l'utilisateur.

Exemples :
- décisions importantes
- idées validées
- informations à retenir
- résumés de discussions

---

### projets/

Conserve les informations de chaque projet.

Chaque projet possède son propre dossier contenant :
- objectifs
- avancement
- tâches
- décisions
- historique

---

### connaissances/

Conserve les connaissances apprises pendant le développement.

Exemples :
- documentation
- recherches
- bonnes pratiques
- ressources utiles

---

## Principes

La mémoire doit être :

- organisée
- facile à retrouver
- évolutive
- fiable
- régulièrement mise à jour

Chaque information importante doit être enregistrée au bon endroit.

Le système de mémoire est conçu pour permettre à Jarvis d'évoluer progressivement en un véritable assistant intelligent.
